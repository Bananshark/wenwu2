# 模型文件路径设置说明

## 当前路径配置位置

模型文件路径在 `index.html` 中有两个地方需要配置：

### 1. AR识别目标文件路径（第53行）

```html
<a-scene mindar-image="imageTargetSrc:./mind/two_wenwu.mind;" ...>
```

**说明：**
- `./mind/two_wenwu.mind` 是MindAR识别文件的路径
- `./` 表示当前目录（index.html所在目录）
- 如果文件在子目录，使用相对路径即可

### 2. 3D模型文件路径（第55-56行）

```html
<a-assets>
  <a-asset-item id="Fe_foot" src="./model/ding.glb"></a-asset-item>
  <a-asset-item id="table" src="./model/铜龙桌子三维模型.glb"></a-asset-item>
</a-assets>
```

**说明：**
- `./model/ding.glb` 是3D模型文件的路径
- `id` 是模型在场景中的标识符
- `src` 是模型文件的实际路径

## 路径设置方法

### 方法一：相对路径（推荐）

**当前目录结构：**
```
新建文件夹 (4)/
├── index.html
├── model/
│   ├── ding.glb
│   └── 铜龙桌子三维模型.glb
└── mind/
    └── two_wenwu.mind
```

**路径示例：**
- `./model/ding.glb` - 当前目录下的model文件夹
- `./model/subfolder/model.glb` - model下的子文件夹
- `../model/ding.glb` - 上一级目录的model文件夹

### 方法二：绝对路径（网络URL）

**示例：**
```html
<a-asset-item id="Fe_foot" src="https://example.com/models/ding.glb"></a-asset-item>
```

**注意：** 使用网络URL需要确保：
- 服务器支持CORS（跨域资源共享）
- 文件可以公开访问
- 网络连接稳定

### 方法三：从配置文件读取（需要代码支持）

可以在 `config.json` 中配置模型路径，然后在代码中动态加载。

## 修改路径步骤

### 步骤1：修改AR识别文件路径

找到第53行，修改 `imageTargetSrc` 的值：

```html
<!-- 原路径 -->
<a-scene mindar-image="imageTargetSrc:./mind/two_wenwu.mind;" ...>

<!-- 修改为其他路径 -->
<a-scene mindar-image="imageTargetSrc:./mind/your_file.mind;" ...>
```

### 步骤2：修改3D模型路径

找到第55-56行，修改 `src` 的值：

```html
<!-- 原路径 -->
<a-asset-item id="Fe_foot" src="./model/ding.glb"></a-asset-item>

<!-- 修改为其他路径 -->
<a-asset-item id="Fe_foot" src="./model/your_model.glb"></a-asset-item>
```

### 步骤3：确保文件存在

修改路径后，确保：
1. 文件确实存在于指定路径
2. 文件名和扩展名正确（.glb 或 .gltf）
3. 文件格式正确

## 常见路径格式

| 路径格式 | 说明 | 示例 |
|---------|------|------|
| `./model/file.glb` | 当前目录下的model文件夹 | 推荐使用 |
| `model/file.glb` | 同上（省略./也可以） | 推荐使用 |
| `../model/file.glb` | 上一级目录的model文件夹 | 跨目录 |
| `/model/file.glb` | 网站根目录的model文件夹 | 绝对路径 |
| `https://example.com/model.glb` | 网络URL | 在线资源 |

## 注意事项

1. **路径区分大小写**
   - Windows系统不区分，但建议保持一致
   - Linux/Mac系统区分大小写

2. **中文文件名**
   - 支持中文文件名，但建议使用英文避免编码问题
   - 如果使用中文，确保文件编码为UTF-8

3. **文件格式**
   - 支持 `.glb` 和 `.gltf` 格式
   - `.glb` 是二进制格式，推荐使用（文件更小）
   - `.gltf` 是文本格式，需要额外的纹理文件

4. **相对路径基准**
   - 相对路径以 `index.html` 所在目录为基准
   - 必须通过HTTP服务器运行，不能直接用file://打开

5. **路径检查**
   - 修改后刷新页面测试
   - 打开浏览器开发者工具（F12）查看控制台错误
   - 检查Network标签页确认文件是否加载成功

## 示例：添加新模型

如果要添加第三个识别目标：

1. **在 `<a-assets>` 中添加模型：**
```html
<a-asset-item id="newModel" src="./model/new_model.glb"></a-asset-item>
```

2. **在场景中添加实体：**
```html
<a-entity mindar-image-target="targetIndex: 2" id="target2">
  <a-gltf-model rotation="0 0 0" position="0 -0.25 0" scale="0.05 0.05 0.05" src="#newModel" animation-mixer></a-gltf-model>
</a-entity>
```

3. **在JavaScript中添加事件监听：**
```javascript
target2.addEventListener('targetFound',()=>showInfo(2));
target2.addEventListener('targetLost',()=>hideInfo());
```

4. **在 `artifactInfo` 中添加介绍信息：**
```javascript
const artifactInfo={
  0:{title:"青铜鼎",content:"..."},
  1:{title:"铜龙桌子",content:"..."},
  2:{title:"新文物",content:"新文物的介绍..."}
};
```

## 故障排除

### 问题1：模型不显示

**检查：**
- 路径是否正确
- 文件是否存在
- 浏览器控制台是否有错误信息
- 文件格式是否正确

### 问题2：路径找不到

**解决：**
- 使用相对路径 `./model/file.glb` 而不是绝对路径
- 确保通过HTTP服务器运行（不能直接用file://打开）
- 检查文件名和扩展名是否正确

### 问题3：中文文件名乱码

**解决：**
- 将文件名改为英文
- 或确保服务器支持UTF-8编码
